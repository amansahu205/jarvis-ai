# Initial Solution Package Checklist

Deadline: April 15, 11:59 PM

## 1. Team Formation and Registration

- Form a team of 4 or 5 UMD students
- Confirm all members are using UMD email credentials
- Register the team
- Store registration details in docs/submission/SUBMISSION_METADATA.md

## 2. Problem Selection

- Choose one provided challenge problem OR define a custom one
- If custom, ensure it is grounded in a real business context
- Ensure complexity is comparable to provided challenge problems
- Document final problem scope in docs/submission/SUBMISSION_METADATA.md

## 3. Data Strategy

- Confirm use of real or synthetic dataset
- If synthetic data is used, document generation approach
- If LLM-generated data is used, record model and prompts at a high level
- Include dataset source and structure notes in reproducibility PDF

## 4. Demo Video (Max 5 Minutes)

- Record a concise walkthrough of the prototype
- Show the problem, key workflow, and output quality
- Upload video to a cloud service
- Verify public access in an incognito browser window
- Add final link to docs/submission/SUBMISSION_METADATA.md

## 5. Reproducibility and Usage PDF (Max 5 Pages)

- Name the file after your team name
- Include setup, run steps, and expected outputs
- Include dataset and environment requirements
- Include troubleshooting section for common failures
- Add PDF link to docs/submission/SUBMISSION_METADATA.md

## 6. Optional Supporting Files

- Add diagrams, result screenshots, architecture notes, and extra docs
- Upload to Google Drive or equivalent
- Add shared link in docs/submission/SUBMISSION_METADATA.md

## 7. Final QA Before Submission

- All links work in incognito mode
- Video is <= 5 minutes
- PDF is <= 5 pages
- Team information is complete and accurate
- Repository README includes a pointer to submission docs
